## Design Context

### Users
Mac users who want to use their physical Mac keyboard to type into an iPhone or iPad. They use the app while pairing a nearby device, switching input routing, and diagnosing Bluetooth or permission issues.

### Brand Personality
Calm, capable, and quietly warm. The interface should create confidence during a technical setup flow, with clear status language and no unnecessary alarm.

### Aesthetic Direction
Bright macOS utility surface inspired by the supplied reference: warm off-white canvas, crisp white panels, restrained orange action accent, green connection states, and a narrow diagnostic rail. Use generous whitespace, subtle borders, soft elevation, and simple line icons. Light mode is the primary theme.

### Design Principles
1. Make connection state immediately legible through one strong status hierarchy.
2. Keep the main job—choose where keyboard input goes—central and low-friction.
3. Use color as status and action semantics: orange for the next action, green for healthy connection, neutral for inactive.
4. Keep diagnostics detailed but visually subordinate to the primary workflow.
5. Preserve a native, trustworthy macOS feel instead of a generic dashboard aesthetic.
